# 学生信息管理系统

## 项目简介

这是一个基于Python开发的学生信息管理系统，提供学生信息的增删改查、数据导入导出、数据备份恢复等功能。系统采用模块化设计，具有良好的可扩展性和可维护性。

## 项目结构

```
学生信息管理系统/
├── data/             # 数据存储目录
│   ├── backups/      # 数据备份目录
│   └── students.json # 学生数据文件
├── logs/             # 日志文件目录
├── src/              # 源代码目录
│   ├── main.py       # 主程序入口
│   ├── managers/     # 管理器模块
│   │   ├── data_manager.py      # 数据管理模块
│   │   └── student_manager.py   # 学生管理模块
│   ├── models/       # 数据模型
│   │   └── student.py           # 学生类
│   ├── ui/           # 用户界面
│   │   └── user_interface.py    # 命令行界面
│   └── utils/        # 工具模块
│       ├── exceptions.py        # 异常定义
│       ├── logger.py            # 日志工具
│       └── validator.py         # 数据验证
├── tests/            # 测试目录
│   ├── test_data_manager.py     # 数据管理器测试
│   ├── test_student.py          # 学生类测试
│   └── test_student_manager.py  # 学生管理器测试
├── create_dirs.py    # 目录创建脚本
├── run.py            # 程序启动脚本
└── run_tests.py      # 测试运行脚本
```

## 系统功能

1. **学生信息管理**
   - 添加学生信息
   - 查询学生信息（按ID、按姓名、按班级）
   - 修改学生信息
   - 删除学生信息
   - 显示所有学生信息

2. **数据导出与导入**
   - 导出学生数据到JSON文件
   - 从JSON文件导入学生数据

3. **数据备份与恢复**
   - 自动备份数据
   - 手动创建备份
   - 从备份恢复数据

4. **统计分析**
   - 学生总数统计
   - 班级数量统计

## 技术栈

- **编程语言**: Python 3.x
- **数据存储**: JSON文件
- **测试框架**: unittest
- **日志系统**: 自定义日志模块

## 使用方法

### 启动系统

在项目根目录下运行：

```bash
python run.py
```

### 运行测试

在项目根目录下运行：

```bash
python run_tests.py
```

## 系统要求

- Python 3.6 或更高版本
- 无需额外依赖包

## 注意事项

1. 数据文件默认保存在`data/students.json`
2. 系统会自动在操作时创建数据备份
3. 日志文件保存在`logs/`目录下
4. 所有操作都有日志记录，方便追踪和排查问题

## 错误处理

系统实现了完善的错误处理机制，主要包括：
- 数据验证错误（如学号格式不正确）
- 学生不存在错误
- 学生已存在错误
- 数据读写错误
- 备份恢复错误

所有错误都会有友好的提示，并记录到日志文件中。# -
